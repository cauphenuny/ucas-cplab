# 死块消除

## CFG 简化 Bug 

### Bug 描述
在 CFG 简化阶段（块合并），存在一个 bug，导致合并基本块时错误地颠倒了指令执行顺序，并错误处理了 Phi 函数，从而导致程序输出不正确。

### 根本原因
当合并线性块（A -> B -> C）时，编译器：
1. **合并指令**： 将 B 和 C 的phi指令和普通指令分别聚合，然后如果B有普通指令，C有phi指令，会颠倒顺序
2. **错误的 Phi 处理**：Phi 函数被保留，但使用了错误的源块，其中计算尚未发生。

### 修复
修改了 `src/backend/ir/optim/dead_block.hpp` 以：
- 确保在块合并时按拓扑顺序追加指令。
- 在合并具有指令的块时，添加对目标块中 Phi 指令的检查。
- 如果源块有指令且目标块以 Phi 指令开头，则阻止合并。
